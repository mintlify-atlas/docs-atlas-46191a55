---
title: "Quickstart"
description: "Connect your first hardware device to the OAHL network in minutes."
---

This guide walks you through installing the OAHL CLI, initializing your node configuration, and starting a running node that agents can reach.

<Steps>
  <Step title="Install the CLI">
    Install the OAHL toolkit globally via npm:

    ```bash
    npm install -g @fredabila/oahl
    ```

    This installs the `oahl` command, which includes the setup wizard, TUI config editor, adapter manager, and the node start command.

    <Note>
      Node.js 18 or later is required. Run `node --version` to check.
    </Note>
  </Step>

  <Step title="Run the setup wizard">
    Run the interactive setup wizard to generate your `oahl-config.json`:

    ```bash
    oahl init
    ```

    The wizard prompts you for:

    - A unique **node ID** (e.g. `my-laptop-01`)
    - Your **provider name** (e.g. `Accra Test Lab`)
    - The **type of hardware** you are connecting first (USB camera, RTL-SDR, mock sensor, or skip)

    When finished, `oahl-config.json` is written to the current directory. The generated file looks like this:

    ```json
    {
      "node_id": "my-laptop-01",
      "cloud_url": "https://oahl.onrender.com",
      "provider_api_key": "123456",
      "plugins": [
        "@oahl/adapter-mock",
        "@oahl/adapter-usb-camera"
      ],
      "provider": {
        "name": "My Personal Test Node"
      },
      "devices": [
        {
          "id": "usb-cam-1",
          "type": "camera",
          "adapter": "usb-camera",
          "capabilities": ["camera.capture"]
        }
      ]
    }
    ```

    <Tip>
      To run your node completely offline without registering with the Cloud Registry, set `"cloud_url": ""` in `oahl-config.json`. The node will still start and local agents can reach it on your LAN.
    </Tip>
  </Step>

  <Step title="Detect connected hardware">
    Before starting the node, scan for connected USB and serial devices to confirm OAHL can see your hardware:

    ```bash
    oahl scan-ports
    ```

    The command identifies connected devices, suggests the matching adapter package, and prints a config snippet you can paste into `oahl-config.json`:

    ```
    1. [USB] USB Video Device
       id: /dev/video0
       status: present
       suggested adapter: @oahl/adapter-usb-camera
       suggested type: camera
       capability hints: camera.capture
    ```

    For scripting or CI, use the `--json` flag to get machine-readable output:

    ```bash
    oahl scan-ports --json
    ```
  </Step>

  <Step title="Start your node">
    <CodeGroup>

    ```bash npm (recommended)
    oahl start
    ```

    ```bash Docker
    docker run -d \
      --name oahl-node \
      -p 8080:8080 \
      -v $(pwd)/oahl-config.json:/app/oahl-config.json \
      --device=/dev/video0 \
      oahl/node:latest
    ```

    </CodeGroup>

    When the node starts successfully, it:

    1. Loads `oahl-config.json` and initializes all adapter plugins
    2. Starts the local HTTP server (default port `3000`)
    3. Registers with the Cloud Registry at the configured `cloud_url`

    <Warning>
      Some adapters (especially serial and BLE) require native C++ bindings. If `oahl start` fails with a build error, use the Docker option — the container ships with all build tools pre-installed.
    </Warning>
  </Step>

  <Step title="Verify the node is healthy">
    With the node running, confirm it is accepting requests:

    ```bash
    curl http://localhost:3000/health
    ```

    A healthy node responds with:

    ```json
    { "status": "ok" }
    ```

    You can also list the capabilities your node is advertising:

    ```bash
    curl http://localhost:3000/v1/capabilities
    ```
  </Step>
</Steps>
